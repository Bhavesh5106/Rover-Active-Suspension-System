brief explanation - The core of this project is moving away from the standard delay() function, which essentially freezes the brain of the Arduino, and instead using a non-blocking approach with millis(). By constantly tracking the time elapsed since the program started, I created three independent clocks that allow each LED to follow its own schedule without waiting for the others to finish. Because the code loops thousands of times every second, it can check these timestamps almost instantly, making the LEDs blink simultaneously at their unique intervals (500ms, 1000ms, and 1500ms). This ensures the timing stays precise and independent, proving that the system can handle multiple real-time tasks at once without any lag or interference.

brief explanation of sensor math used - For this project, Instead of looking at a clock, the logic focuses on the Relative Timing between two events—essentially calculating the reaction gap using the formula t_differce = t_finish - t_start. This measures the exact moment the LED pops on versus the millisecond the button is pressed. To keep the hardware simple and the signals clean, I used INPUT_PULLUP logic; this keeps the pin at a High voltage by default and only triggers the calculation when the button is pushed to Ground. Finally, to make sure the game isn't predictable, I used randomSeed(analogRead(0)) to sniff atmospheric noise from an empty pin, which creates a truly random start every time so you can't just memorize the pattern.
